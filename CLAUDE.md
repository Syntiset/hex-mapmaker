# CLAUDE.md — Hex Map Maker (GURPS)

Проектный свод правил для AI-агента. Глобальные правила из `~/CLAUDE.md` (Karpathy guidelines) сохраняют силу — этот файл их дополняет.

## Что это за проект

Web-редактор гексагональных карт для GURPS. Стиль по умолчанию — Fallout (пустоши, руины, радиация). Стек: **Vite + React + TypeScript + react-konva + zustand**. Сохранение карт — JSON, экспорт — PNG. Пользователь не программист.

## Команды

- Установка: `npm install`
- Dev: `npm run dev` (Vite, по умолчанию http://localhost:5173)
- Сборка: `npm run build` (включает `tsc -b`)
- Lint: `npm run lint`
- Typecheck отдельно: `npx tsc -b --noEmit`

## Definition of Done

Перед завершением задачи:
1. `npm run build` проходит без ошибок (это автоматически даёт typecheck).
2. Если правил визуальную часть — открыть `npm run dev` и проверить вручную.
3. Обновлены затронутые файлы в `docs/agent/` (как минимум `00_current_state.md`, `04_changelog.md`, `07_handoff.md`).
4. Активный план в `docs/agent/03_active_plan.md` синхронизирован: завершённые пункты убраны.

## Канонические пути

- Hex math (единственный источник истины по координатам): `src/hex/hex.ts`
- Стейт карты (zustand): `src/store/mapStore.ts`
- Палитры тайлов: `src/tiles/<setting>.ts` (на старте — `fallout.ts`)
- Главный канвас: `src/components/HexGridCanvas.tsx`
- IO (save/load/export): `src/io/`
- Документация состояния: `docs/agent/`

## Соглашения

- Язык документации и UI — русский. Технические идентификаторы (имена файлов, команды, ключи JSON) — в оригинале.
- Гексы — **pointy-top**, axial-координаты `(q, r)`. Любая новая работа с координатами идёт через `src/hex/hex.ts`, не дублировать.
- JSON-схема карты версионируется (`{ version, ... }`); при breaking change — поднять версию и добавить миграцию.
- Новый сеттинг тайлов = новый файл в `src/tiles/`, реализующий тот же интерфейс `TileDef[]`. Никаких хардкодов сеттинга в компонентах.
- Не добавлять зависимости без явной причины. Konva и zustand — основные; перед новой либой подумать дважды.
- Surgical changes: править только то, что относится к задаче. Никаких попутных рефакторингов чужого кода.

## Чего не делать

- Не использовать сторонние hex-библиотеки — math простой, держим свой.
- Не вкомпиливать художественные ассеты в репозиторий до явной просьбы. Все тайлы сейчас — процедурные плейсхолдеры.
- Не превращать `CLAUDE.md` в энциклопедию. Подробности — в `docs/agent/`.
