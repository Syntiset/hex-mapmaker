## Конкретный план насыщения

### Шаг 1 — Процедурная текстура биома (приоритет: ВЫСОКИЙ)

`src/render/biomeSprite.ts`, проход 1 (текстура).

**Сейчас:** `stippleLayer` рассыпает кружочки случайных цветов поверх плоской заливки.

**Стало:** попиксельный fBm + domain warp поверх базового fill, модулирующий **lightness ±10-15%** (HSL-сдвиг или multiply с серой картой). Поверх — turbulence-маска для пятен (грязь, выгоревшие участки).

```ts
// псевдокод
for (each pixel in hex bbox) {
  const t = fbm(domainWarp(noise2D, x/scale, y/scale, 0.25), 4 octaves);
  imageData[i] = baseRGB * (1 + t * 0.15);
}
```

**Эффект:** биом перестаёт быть «заливкой одним цветом», получает живую неоднородность.

### Шаг 2 — Composite-слои (приоритет: ВЫСОКИЙ)

`src/render/biomeSprite.ts`, реструктура passes:

```
1. base fill              source-over
2. noise albedo           multiply       (из шага 1)
3. edge vignette          multiply       (радиальный градиент к краям, alpha 0.3)
4. directional highlight  screen         (псевдо-light, смещённый радиальный grad)
5. color tint             soft-light     (атмосферный warm/cool)
6. glow (optional)        screen         (radiation/fire — оставить как есть)
7. icon                   source-over    (без изменений)
```

**Эффект:** глубина без новых ассетов. Pipeline стабильный, не зависит от типа биома.

### Шаг 3 — Rough.js контур гекса (приоритет: СРЕДНИЙ)

`src/render/drawHex.ts`. Заменить чистый `ctx.stroke(hex)` на `rc.path(hexSvgPath, { roughness: 0.5-0.7, seed: hash3(q,r) })`.

**Эффект:** граница перестаёт выглядеть «cad-овской», получает рукописный характер. Чуть растёт ощущение «нарисовано рукой».

### Шаг 4 — Rough.js hachure для биомов (приоритет: СРЕДНИЙ)

Отдельный pass с `ctx.clip()` по гексу. По биомам:
- wasteland → `hachure` 45°
- swamp → `zigzag`
- irradiated → `dots`
- mountain/cliff → `cross-hatch`
- water → без hachure (есть ripples)

**Эффект:** биомы приобретают «характер» поверхности. Риск: если переборщить — каша.

### Шаг 5 — Иконки (приоритет: НИЗКИЙ)

Без изменений. Оставляем handcrafted, как сейчас.

## Риски

| Риск | Серьёзность | Митигация |
|---|---|---|
| 160k simplex-вызовов на бейк (4 окт. × 40k пикс.) | низкая | ~2 мс при бейке, runtime не страдает |
| Rough hachure вылезает за hex | средняя | обязательно `ctx.save() → clip → ... → ctx.restore()` |
| `multiply` на прозрачном = чёрный | средняя | непрозрачный base fill первым слоем |
| Rough seed нестабилен между версиями | низкая | LRU инвалидируется при деплое — приемлемо |
| Бандл +13 KB | низкая | — |
| Шаги 1-2 ломают визуал биомов до перенастройки | средняя | делать пошагово с screenshot review |

---

## Рекомендация по порядку

**Делать поэтапно, с ревью после каждого шага:**
1. Шаг 1+2 (noise + composite) — самый большой эффект, делать первым
2. Скриншот → если зашло, шаг 3 (rough контур)
3. Шаг 4 (hachure) — последним, может оказаться излишним

**НЕ начинать с rough.js** — без насыщенного фона он не сработает, выйдет тонкая обводка на плоском фоне.
