## Resolved
### ISSUE-003 — Sample-vs-clip mismatch при wavy-границе биомов (ИСПРАВЛЕНО v1.0.5+rework, 2026-05-07)
**Симптом:**
Стиппл и декорации (точки, травинки, камешки) на стыке двух биомов выглядели «обкусанными» / разреженными. На wavy-границе образовывалась видимая светлая полоса — зона без stipple-затемнения. При добавлении биома рядом текстура существующего ВНЕЗАПНО изменялась — появлялись «дыры» вдоль нового шва.

**Корневая причина:**
Wavy-граница смещается **двунаправленным** 2D-вектором шума `noiseVec(x, y)`. Соседние гексы согласованно сэмплируют шум в одной и той же мировой точке — поэтому на shared edge их wavy-полигоны идеально совпадают (фикс v1.0.3). Но смещение **асимметрично**: одна сторона теряет территорию, другая получает.

При этом стиппл/декорации сэмплировались в **clean-hex форме** (или с margin'ом), а **клипались displaced wavy полигоном**. Это создавало две зоны разрежения на каждом ребре:
- **Зона A** (wavy ушёл ВНУТРЬ гекса): точки сэмплированы в clean-hex, но клип их выкинул → пустота
- **Зона B** (wavy ушёл НАРУЖУ в соседа): clean-hex до этого участка не достаёт → точек никогда не было → пустота

У соседнего гекса картина зеркальная: его «зона A» = моя «зона B» и наоборот. Итог: оба гекса оставили пустоту в одной и той же зоне → светлая полоса на wavy-шве.

**Симптомные коммиты-попытки (откачены через reset до ab1ad98):**
v1.0.6, v1.0.7, v1.0.8, v1.0.9, v1.0.10, v1.0.11, v1.0.12, v1.0.13, v1.0.14, v1.0.15.
Эти попытки боролись со СЛЕДСТВИЕМ — крутили margin'ы, переносили рендер из спрайта в runtime и обратно, добавляли halo и т.д. Каждая исправляла одно проявление, но переносила баг в другое место.

**Правильный фикс:**
Сэмплировать стиппл/декорации **прямо внутри displaced wavy полигона** через rejection sampling. Sampling matches clipping → плотность равномерна по wavy-форме автоматически.

Реализация:
- `pointInPolygon(px, py, polygon)` в `src/render/displaced.ts` — ray-casting test
- `sampleInWavyPolygon(seed, idx, cx, cy, size, polyAbs)` в `src/render/drawHex.ts` — rejection sampling в bbox с тестом против полигона
- `sampleArea(...polyAbs?)` — унифицированный entry point: с polyAbs использует полигональный sample, без — fallback на safe-hex margin
- `drawHexTexture` / `stippleLayer` / `drawDecoration` принимают опциональный `polyAbs`
- В `biomeSprite.buildSprite` вычисляется `polyAbs = polyWorld.map(p => ({x: p.x+dx, y: p.y+dy}))` и пробрасывается в `drawHexTexture`

Палитровые превью (`drawBiomeRich`) и tile decoration в clean hex clip продолжают использовать safe-hex margin sampling — там нет wavy-полигона.

**Урок для будущего:**
Если текстура клипается полигоном P, она должна и сэмплироваться внутри P. Любой mismatch (sample в S, clip к P, S ≠ P) даёт неравномерную плотность в зоне различия S и P. Margin'овые компромиссы НЕ ИСПРАВЯТ эту фундаментальную проблему — они только сдвинут визуальные артефакты.

**Дополнение (2026-05-07):**
После фикса plотности всё равно оставался визуальный изъян: wavy-граница АСИММЕТРИЧНА — biome A bulge'ил в clean-hex территорию biome B и наоборот. Технически без overlap (тайлинг идеальный), но воспринималось как «биомы наезжают друг на друга».

Корень асимметрии: `noiseVec` возвращает 2D-вектор в [-1, +1]², смещение могло идти В ЛЮБУЮ сторону. Один shared wavy line одновременно даёт «отъедание» одному биому и «расширение» другому.

**Inward-only попытка:** wavy-смещение было сделано направленным к центру каждого гекса (`|noise| * sin(πt) * AMPLITUDE`). Биом строго внутри своего clean-hex, bulge'ов наружу нет. Но при `AMPLITUDE = 0.30` появлялись видимые «выгрызы» между соседями (BG просвечивал в зазорах). Снижение до `0.10` дало едва заметную волнистость — практически ничего не давало визуально, но всё равно требовало displaced полигоны и связанную сложность.

**ФИНАЛЬНОЕ РЕШЕНИЕ (v1.1.0):** wavy/displaced функционал **удалён целиком**. Файл `src/render/displaced.ts` стёрт. Все sample-функции (`sampleInSafeHex`, `sampleInWavyPolygon`, `sampleArea`) удалены. Stipple/decoration вернулись к простому rectangular sampling в bbox вокруг центра гекса; clean hex clip обрезает естественно.

Soft blending между биомами теперь делает **runtime `drawBiomeBlob`** — radial gradient с alpha falloff на радиусе 1.25*size. «Хвост» blob'а перекрывается с blob'ами соседей, давая мягкие цветовые переходы оптически (alpha-blend), а не геометрически (wavy clip). Это работает без побочек:
- Нет sample-vs-clip mismatch (sampling всегда в clean hex, clip в clean hex)
- Нет двунаправленной асимметрии (blob симметричен по гексу)
- Соседние blob'ы перекрываются по alpha, не «откусывая» друг друга

**Урок:** прежде чем городить сложную геометрию (displaced polygons, rejection sampling в полигонах, multi-pass clipping), стоит проверить, можно ли получить тот же визуальный эффект через alpha-blending простых runtime gradient'ов. Wavy выглядела заманчиво ради органичного вида карт, но вытащила за собой каскад проблем; альфа-перетекание blob'ов даёт похожий «мягкий стык» гораздо проще.

Lex-каноникализация, `pointInPolygon`, `sampleInWavyPolygon`, `EDGE_TO_NEIGHBOR_OFFSET`, neighborMask вычисление в drawScene — всё удалено за ненадобностью.

## Medium
### ISSUE-001 — Производительность на больших картах
- Симптом: при картах ~5000+ гексов возможны просадки FPS при pan/zoom.
- Где проявляется: `HexGridCanvas`.
- Причина: каждый гекс — отдельная Konva-нода без кэширования.
- Временный обход: использовать карты до 60×60.
- Следующий шаг: при появлении реальной нагрузки — `Layer.cache()` для статичных слоёв либо переход на pure-canvas рендер.
